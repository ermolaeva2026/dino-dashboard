# Автообновление дашбордов PRK-2026-DS

Источник статусов реализации: `PRK-2026-DS_KSG_DinoPark.mpp`.

Источник фото: `Фотофиксация работ`.

Источник сметы и документов: корень проекта `ДиноПарк. Модернизация`.

Запуск вручную:

```powershell
powershell -ExecutionPolicy Bypass -File "Автообновление дашбордов\update-dashboards.ps1"
```

Или двойной клик по `update-dashboards.cmd`.

Для ручного запуска канонической пересборки можно использовать:

```cmd
rebuild-canonical-dashboards-scheduled.cmd
```

Фактически wrapper запускает:

```powershell
rebuild-canonical-dashboards-scheduled.ps1
```

Логи пишутся в `Автообновление дашбордов\logs`.

`update-dashboards-scheduled.cmd` оставлен как технический wrapper для проектного скрипта, но не должен быть основой расписания: он не является владельцем эталонной структуры основного табличного дашборда.

Что обновляется:

- `PRK-2026-DS_Рабочий_дашборд.html` — даты, проценты, статусы задач, блок "требуют актуализации", документы.
- `PRK-2026-DS_Dashboard.html` — фазы из КСГ и фотогалерея из папок фотофиксации.
- `PRK-2026-DS_dashboard-data.json` — машинный снимок данных для проверки.

Активное автоматическое расписание держится на двух канонических задачах Windows Task Scheduler:

- `PMO - ДиноПарк дашборд (PRK-2026-DS)` — основной дашборд `PRK-2026-DS_Dashboard.html`.
- `PMO - ДиноПарк РАБОЧИЙ дашборд (PRK-2026-DS)` — рабочий дашборд `PRK-2026-DS_Рабочий_дашборд.html`.

Обе задачи запускаются ежедневно в `00:00` и `18:00`.

Публикация:

- `ДиноПарк — публикация на GitHub Pages` — ежедневно в `00:10` и `18:10`.

Отключены, потому что запускали не тот wrapper и могли ломать структуру:

```cmd
PRK-2026-DS Dashboards 09
PRK-2026-DS Dashboards 18
```
